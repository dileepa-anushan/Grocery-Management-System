🛒 GroceryManagement — Online Grocery Management System
A full-stack Java EE web application built with Servlets, JSP, and the DAO pattern for managing online grocery orders, inventory, and transactions.


## 📋 Table of Contents - [Introduction](#-introduction) - [Key Features](#-key-features) - [Tech Stack](#-tech-stack) - [System Architecture](#-system-architecture) - [User Roles & Access](#-user-roles--access) - [Data Models & DAOs](#-data-models--daos) - [Servlet Structure & Endpoints](#-servlet-structure--endpoints) - [Prerequisites](#-prerequisites) - [Project Setup](#-project-setup) - [Running the Application](#-running-the-application) - [Project Structure](#-project-structure) --- ## 🚀 Introduction **GroceryManagement** is a Java web application designed to automate and streamline online grocery shopping and store operations. Built using Java Servlets, JSP, and standard Enterprise Web Architecture, it provides an intuitive e-commerce experience for customers and a control panel for administrators to manage stock, orders, and sales history. --- ## ✨ Key Features | Module | Description | |---|---| | **User & Auth Management** | Secure user registration, authentication, session handling, and role-based portal routing. | | **Product Catalog & Filtering** | Browse categorized items (Fruits, Dairy, Fresh Products), with product sorting and review displays. | | **Cart & Checkout Workflow** | Interactive shopping cart handling, item updates, cost calculation, and checkout validation. | | **Order Queue & Processing** | Asynchronous queue-based order processing (`OrderProcessingService`, `OrderQueue`) with status tracking. | | **Inventory Control** | Real-time stock level monitoring and admin inventory updating tools. | | **Payments & Transactions** | Payment validation, transaction logging, and history archiving (`PaymentDAO`, `TransactionDAO`). | | **Customer Reviews** | Product rating and feedback mechanism with content sanitization/validation. | | **Admin Dashboard** | Oversight panel for system stats, pending orders, inventory warnings, and transaction logs. | --- ## 👥 User Roles & Access | Role | Access Path | Capabilities | |---|---|---| | **Customer** | `/home`, `/products`, `/cart` | Browse catalog, filter items, add to cart, checkout, view order status, leave reviews | | **Admin** | `/admin-dashboard`, `/inventory` | Manage catalog, modify stock levels, review global transactions, process order queues | --- ## 🗃️ Data Models & DAOs The application follows strict object-relational encapsulation using Data Access Objects (DAOs): | Model | DAO | Key Purpose | |---|---|---| | `User` | `UserDAO` | Authentication and account management | | `Product` | `ProductDAO` | Catalog item metadata, pricing, and category mapping | | `Cart` | `CartDAO` | Session/persistent cart state management | | `Order` / `OrderQueue` | `OrderDAO` | Order lifecycles and queue processing | | `Inventory` | `InventoryDAO` | Stock quantities and replenishment flags | | `Payment` / `PaymentDetails` | `PaymentDAO` | Payment verification and details mapping | | `Transaction` | `TransactionDAO` | Financial audit trail and history logs | | `Review` / `ReviewDisplayModel` | `ReviewDAO` | User ratings and testimonial rendering | --- ## 🔌 Servlet Structure & Endpoints | Endpoint | Servlet Class | Description | |---|---|---| | `/home` | `HomeServlet` | Store landing page and featured categories | | `/user` | `UserServlet` | Login, registration, and user profiles | | `/products` | `ProductServlet` | Catalog browsing, filtering, and sorting | | `/cart` | `CartServlet` | Item addition, quantity updates, and cart viewing | | `/order` | `OrderServlet` | Order creation, processing, and summary | | `/payment` | `PaymentServlet` | Payment submission and transaction processing | | `/inventory` | `InventoryServlet` | Admin inventory adjustments | | `/admin-dashboard` | `AdminDashboardServlet` | Central admin operations center | | `/review` | `ReviewServlet` | Feedback submission and verification | --- ## 📦 Prerequisites Before running the application, make sure you have: - **Java Development Kit (JDK)** >= 8 (11 or 17 recommended) - **Apache Maven** >= 3.6.x - **Apache Tomcat Server** >= 9.x or 10.x - **IDE**: IntelliJ IDEA / Eclipse (with Smart Tomcat plugin support) --- ## ⚙️ Project Setup ### 1. Extract / Clone Repository ```bash git clone https://github.com/your-username/grocerymanagement.git cd grocerymanagement ``` ### 2. Build the Project Compile the Java classes and package the application using Maven: ```bash mvn clean package ``` --- ## ▶️ Running the Application ### Option A: Using Smart Tomcat (IntelliJ IDEA) 1. Open the project in **IntelliJ IDEA**. 2. Add a new Run/Debug Configuration -> Select **Smart Tomcat**. 3. Set **Deployment Directory** to `src/main/webapp`. 4. Set **Context Path** to `/grocerymanagement`. 5. Run the configuration and open `http://localhost:8080/grocerymanagement/home`. ### Option B: Deploying WAR to Apache Tomcat 1. Copy the generated `.war` file from `target/` into Tomcat's `webapps/` folder. 2. Start Tomcat using `bin/startup.sh` or `bin/startup.bat`. 3. Open `http://localhost:8080/grocerymanagement`. --- ## 📁 Project Structure ``` grocerymanagement/ ├── pom.xml # Maven dependencies & build config ├── src/ │ └── main/ │ ├── java/com/grocerymanagement/ │ │ ├── config/ # FileInitializationUtil │ │ ├── dao/ # Data Access Layer (8 DAOs) │ │ ├── dto/ # PaymentDetails, Data Transfer Objects │ │ ├── model/ # Domain Entities (User, Product, Order, etc.) │ │ ├── service/ # Business Logic Services │ │ ├── servlet/ # Http Controllers (11 Servlets) │ │ └── util/ # Security, Validation & Converters │ └── webapp/ │ ├── assets/ # Images (fruits, dairy), CSS (dark-theme.css) │ ├── WEB-INF/ # Web deployment descriptors │ └── *.jsp # Frontend View templates └── README.md ```
